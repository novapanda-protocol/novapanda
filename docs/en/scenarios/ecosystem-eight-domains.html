Ecosystem · Roles & boundaries

Who plays which role

Four core roles and four foundations — unify delivery facts, not operating systems.

Core roles

1

Agent · exchange party

Holds Ed25519 identity; the real subject of propose / deliver / dual-sign.

2

Node · operated instance

Orchestrates exchanges, quotas, APIs — substitutable, not the protocol.

3

Registry · trust registry

Public registration of rules, ontologies, and extension packs.

4

Settler · clearing partner

Optional rails (mock / x402 / fiat) — downstream translation only.

Foundations

5

SPEC

Single source of truth for state machine & semantics — CC BY 4.0 commons.

6

Manifest

/.well-known/novapanda.json — stranger agents discover capabilities instantly.

7

VDC

Verifiable Delivery Credential — dual-signed, canonical, offline re-verifiable.

8

Verifier

Local or any impl can reverify — compatibility by vectors, not domains.
