Eight domains: how deliverables attach

API, MCP, Skill, workflows, OTA, P2P… a broad ecosystem does not cram everything into one state machine — it uses one verifiable delivery grammar with layered extensions. NovaPanda is a delivery recognition layer, not an OS for everything.

Three sentences

Unify delivery facts (VDC), not workflows, OTA, or flight control.

Small core, wide extensions: Profiles · bindings · registries stack capability.

Compatibility by vectors, not “must pass our node.”

Layers

Narrative (this page, scenario catalog)
Body (zero node: quota · audit · sandbox · Operator)
Bindings (HTTP · MCP · A2A · Skill · Proxy)
Capabilities (MIN · NODE · BUNDLE · PHYS · SETTLE · DELEGATE · PRIV · LITE)
Registries (ontology · rules · extension packs)
CORE (identity · VDC · exchange state machine · reverify)

Eight domain cards

① Remote services

  • wired HTTP API
  • wired MCP · BINDING-MCP
  • in progress ToolBundle
  • adjacent Remote Pod · Proxy

② Local execution

  • in progress Skill · inference packs
  • adjacent Script · firmware · diagnostic probes

③ Orchestration

  • in progress Workflows (client-side Composer)
  • adjacent Task sharding · event handling

④ No-code rules

  • wired Exchange state machine · quota
  • in progress Triggers/audit · prompt templates

⑤ Developer tools

  • wired Schema spec
  • in progress SDK · CLI (reverify)

⑥ Delivery upgrades

  • not in core OTA (evidence hash only)
  • in progress Extension packs

⑦ Security & access

  • wired Delegation NP-DELEGATE
  • in progress Access policy packs

⑧ Mesh collaboration

  • in progress Delta sync NP-LITE
  • adjacent P2P mesh / federation
