Intelligent Open Delivery Protocol · Constitution

Open delivery grammar for the intelligent world

Any agent and device can exchange value across organizations with no pre-existing relationship — no minting, no fund custody, no protocol fee. Truth in delivery is a credential (VDC) anyone can re-verify offline.

Litmus · consistency is membership

Can a stranger agent complete delivery without our proprietary stack and without permission, per the public spec — and can anyone re-verify? Yes → NovaPanda-compatible; no → not the same protocol.

What we solve

Payment is downstream translation; delivery records are first-class.

Autonomous agents and devices will collaborate at scale. They are strangers under different owners yet must exchange value. NovaPanda does not answer “which coin to pay with” — it answers: how to leave a verifiable delivery record — structured, dual-signed, offline-verifiable, independent of price, chain, or platform.

“We may be strong, but we must be substitutable and exitable” — proof of a healthy network and open protocol.

Five principles

The soul of the project — protection of the mission, not pedantry.

1

Open and equal

The protocol is public capability for the intelligent world — not a moat for one model vendor. Permissionless implementation, self-hosted nodes, independent VDC reverify.

2

Never mint

No globally tradable protocol coin. mock / x402 / fiat are pluggable settlement rails — not issuance.

3

No protocol rent

Norms belong to the commons. Company value is in operations, liquidity, and service — not a toll on every exchange.

4

Prove delivery, then price

VDC and deterministic acceptance are first-class; price and settlement follow. Trial validates delivery grammar, not payment rails.

5

Soul vs body

Soul (spec, VDC format, state machine) is a public good, CC BY 4.0; body (hosted nodes, clearing networks) is operated by the company, but anyone can run an equivalent node. Nodes are substitutable; truth (VDC) is not monopolizable.

Access surfaces

SPEC §7 requires equivalent entry points. Rule: surfaces only translate — MUST NOT change credentials, state machine, or reverify semantics.

Skill · MCP · A2A

For agent ecosystems (Cursor, MCP hosts, A2A runtimes). Register tools or Agent Cards without mastering signing details.

SDK · adapter · HTTP

Python + TypeScript SDKs; ProviderAdapter for existing systems; any language via HTTP + local signatures. Plugfest uses heterogeneous stacks.

Interop comes from following the spec, not “everyone uses our SDK”. See surface table and novapanda/surfaces/.

  1. Protocol, norms, and trademark intent are public goods — not owned by one company forever.

  3. Governance changes must not break re-verifiability or backward compatibility of existing VDCs.
