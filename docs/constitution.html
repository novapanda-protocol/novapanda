协议宪法

这是 NovaPanda 的公共原则与分层归属。完整技术规范见 SPEC.md；治理流程见 GOVERNANCE.md。

我们在解决什么

智能体与智能设备要在无预建关系下交换价值。价值交换的第一公民不是支付，而是 VDC（可验证交割凭证）——结构化、双签、任何人可脱离单一平台独立复验。

Litmus 测试：陌生 Agent 能否不依赖我方专有组件、不经许可，照规范完成交割并被任何人独立复验？

五条铁律（不可动摇）

1. 开放、平等协议是公共能力；任何人可无许可实现、自建节点、独立复验 VDC。
2. 永不铸币协议层不造全球流通、可投机的货币。清算轨可插拔，不等于发币。
3. 永不对协议抽成规范归共域；公司价值在运营与服务质量，不在协议租金。
4. 先证交付，后谈价格VDC 与验收是一等公民；mock / x402 / 法币只是可选下游清算。
5. 灵魂与身体分离规范与商标意图归中立公共物；公司是可替代的托管与运营主体——节点可被替代，真理（VDC）不可被垄断。

整体层级（你在哪里）

L0 身份
Ed25519 Agent 身份
私钥仅存本地，不上送任何节点。
L1 凭证
VDC · 双签 · 离线复验
交割的「真理」；与价格、链、平台无关。
L2 协议
状态机 · 验收规则 · 资源本体
开源公共物（CC BY 4.0）。任何人可实现。
L3 节点
交换编排 · 信誉账本 · HTTP API
参考实现开源（Apache-2.0）；人人可自建同构节点。
L4 清算
Mock · x402 · AP2 · 法币伙伴
可插拔下游；Trial 仅用 mock，无真钱。
运营
零号节点 node.novapanda.io
我们运营的公开 mock 试用实例之一；不是协议本身，可被替代。

治理铁律（摘自 GOVERNANCE）

  1. 协议 / 规范 / 商标意图是公共物，不归属单一公司。
  2. 任何人可不经许可实现规范、自建节点、接入交换。
  3. 治理变更不得破坏既有 VDC 的可复验性与向后兼容路径。
