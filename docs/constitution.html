Protocol Constitution · 公共原则

智能世界的交割语法

让任意智能体与智能设备，在无预建关系下完成跨主体价值交换—— 不发币、不托管资金、不对协议抽成；交割的真理是一份 任何人都能脱离单一平台、独立复验的凭证（VDC）。

不铸币不绑链协议归共域 公司只做身体验收可复验凭证可退出
Litmus · 一致性即成员资格

陌生 Agent 能否不依赖我方专有组件、不经许可， 照公开规范完成交割，并被任何人独立复验？ ——能，即 NovaPanda-compatible；不能，即不是同一协议。

我们在解决什么

支付是下游翻译；交付记录才是第一公民。

未来是大量自主智能体与设备协作的世界。它们彼此陌生、隶属不同主体，却需要交换价值。 NovaPanda 不回答「用什么币付钱」，而回答：如何留下一份可验证的交付记录—— 结构化、双签、可离线验真，与价格、链、平台无关。

「我强，但我可被替代、可被退出」——这证明网络健康，也是协议开放性的最高表达。

五条铁律

项目的魂。不是洁癖，是对使命的保护。

1

开放、平等

协议是智能世界的公共能力，而非补强某家大模型的围墙。无许可实现、自建节点、独立复验 VDC。

2

永不铸币

协议层不造全球流通、可投机的货币。mock / x402 / 法币是可插拔清算轨，不等于发币。

3

永不对协议抽成

规范归共域。公司价值在运营、流动性与服务质量，不在对每笔交换收「协议租金」。

4

先证交付，后谈价格

VDC 与确定性验收是一等公民；价格与清算排在后面。Trial 验证的是交割语法，不是支付能力。

5

灵魂与身体分离

灵魂（规范、VDC 格式、状态机）归公共物，CC BY 4.0； 身体（托管节点、清算网络）由公司运营，但人人可自建同构节点。 节点可替代，真理（VDC）不可被垄断。

整体层级

从下往上：身份 → 凭证真理 → 开源协议 → 开源节点 → 可选清算 → 运营实例。

L0 身份
Ed25519 · 私钥不上送
Agent 在本地持有身份；节点只验签，不托管密钥。
L1 凭证
VDC · 可验证交割凭证
交割的「真理层」——双签、canonical、任何人可离线 reverify。
L2 协议
状态机 · 规则 · 资源本体
开源公共物（CC BY 4.0）。SPEC.md
L3 节点
交换编排 · 信誉 · API
参考实现开源（Apache-2.0）。人人可部署，一致性套件 C1–C7 互验。
L4 清算
Mock · x402 · AP2 · 法币伙伴
下游可插拔；公开 Trial 仅用 mock，无真实资金。
运营
零号节点 · node.novapanda.io
我们运营的公开 mock 实例之一——不是协议本身，可被任何同构节点替代。

治理承诺

  1. 协议、规范、商标意图是公共物，最终不归属单一公司。
  2. 任何人可不经许可实现规范、自建节点、接入交换。
  3. 治理变更不得破坏既有 VDC 的可复验性与向后兼容路径。
