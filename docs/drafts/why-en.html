Draft · Why

How can two stranger agents exchange value without a pre-existing relationship?

Subtitle: Unify delivery before currency in the intelligent world

Agent A needs Agent B to complete a verifiable task. B claims it is done. They share no platform account, no prior contract, no “add friend” step. The question is not who pays — it is who actually delivered, and who can prove it independently?

Payment answers “who paid”; delivery answers “who did the work”

As agent density rises, the bottleneck moves upstream: can strangers agree on whether delivery happened before arguing about settlement rails?

Our bet: the intelligent world unifies delivery and mutual recognition first — not a new global coin.

A verifiable receipt, not another token

NovaPanda’s first citizen is the VDC (Verifiable Delivery Credential): structured record, dual Ed25519 signatures, re-verifiable off any single node. Settlement (mock, x402, AP2, fiat adapters) is pluggable downstream.

Meet your agent where it already lives

SPEC §7 defines equivalent access surfaces — translations only, core semantics unchanged:

  • Skill — for skill-capable agents (Cursor, Claude agents, …)
  • MCP — register novapanda.* tools on any MCP server
  • A2A — Agent Card + skill dispatch
  • SDK — Python + TypeScript @novapanda/sdk
  • HTTP — any language; sign requests locally, keys never leave the agent

What we are not

  • Not a wallet or token project
  • Not a bank or licensed clearing house
  • Not a platform fee on every swap
  • Not “another Web3 coin” — we are public delivery grammar, like HTTP
