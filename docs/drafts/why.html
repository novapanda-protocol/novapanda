两个陌生 Agent，如何在没有预建关系的情况下交换价值？

副标题：智能世界先统一交割，再统一货币

想象两个互不相识的软件 Agent：A 需要 B 完成一项可验收的任务，B 声称已经完成。 它们没有共同的平台账号，没有历史合同，也没有「加好友」这一步。 问题不是「谁付钱」，而是：谁真的交付了？谁能独立证明？

支付解决「谁付钱」，交割解决「谁真的做了」

人类商业世界里，我们习惯先建立关系——公司、合同、账户、发票——再谈交换。 支付系统在此基础上运转：它记录资金的转移，却很少成为交付本身的真理。

智能体密度上来之后，瓶颈会前移： 两个陌生主体能否在无预建关系下，就「是否完成交付」达成一致？ 这比「用哪种币结算」更基础。

我们的判断是：智能世界不是先统一货币，而是先统一「交割与互认」。 成功标志不是清算额，而是：陌生对端能否互认一张可独立验真的交付收据。

一张可验真的收据，而不是又一种钱

NovaPanda 的第一公民是 VDC（可验证交割凭证）： 结构化记录、双方 Ed25519 双签、任何人可脱离单一平台复验。 它回答的是：这件事有没有按约定完成？——而不是 钱从哪到哪。

清算（mock、x402、法币通道）是可插拔的下游。 你可以用任何支付方式结算，但交付的真理不应绑死在某个 App 或某条链的账本里。

Litmus

一个从没见过的、用别的语言 / 框架 / 结算方式的陌生 Agent，能否 不依赖我方专有组件、不经我方许可，照公开规范完成交割， 并被任何人独立复验？

今天就能排练：陌生软件 Agent 的模拟舱

完整路径并不遥远，今天就可以在公开 mock 节点上跑通：

propose → contract → escrow → deliver → verify → confirm → SETTLED

两个事先互不认识的主体，产出一份双签 VDC。 这是未来陌生车、桩、机器人之间交换的替身排练—— 同一套资源本体、状态机与凭证格式，日后可平移到物理世界。

  • 试用：开发者 Trial · node.novapanda.io（mock，无真钱）
  • 复验：novapanda.reverify — 脱离节点独立验真

我们不是什么

为了避免误读，先说清楚边界：

  • 不是钱包，不是发币项目
  • 不是银行、持牌清算机构，Trial 不涉及真实资金
  • 不是对每笔交换抽成的平台
  • 不是「Web3 又一种币」——我们做的是公共交割语法，像 HTTP，不属于任何公司

路径：从软件到万物

北极星是智能万物交换：一切智能体、设备、大模型， 在无预建关系下用同一套可验证交割语言彼此交换价值。 路径务实——今天从陌生软件 Agent 出发，明天平移到能源、机器人与合规物理交割。 详见 愿景与路径。
