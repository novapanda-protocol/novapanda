开发者 Trial

在验证协议与 VDC，不是在验证支付。先读 协议宪法了解层级与原则，再按下列步骤连 零号 mock 节点。

mock 试用边界

✓ 完整状态机 + 双签 VDC + 离线复验  |  ✗ 真钱 / 稳定币 / 持牌清算

官网 vs 零号控制台

novapanda.io 负责介绍、看图、接入说明 — 无需注册。 动手试 API 用 node.novapanda.io；只有需要提额或「我的交换」时，才在控制台内注册运营账户。

Step 0 · Cursor / Codex / MCP（可选）

若你已在用 Cursor、Codex、Windsurf、Cline 等 MCP IDE：在 MCP 配置中加入 NovaPanda 工具集（novapanda.*）， 指向 https://node.novapanda.io。读工具走 MCP；写路径（propose / deliver 等）仍须本地 Agent 签名。 见 智能开发工具对接 · BINDING-MCP · novapanda/bindings/mcp_tools.py。

Step 1 · 检查节点

curl -fsS https://node.novapanda.io/health
curl -fsS https://node.novapanda.io/.well-known/novapanda.json

Step 2 · Python 一键 SETTLED

git clone https://github.com/novapanda-protocol/novapanda.git
cd novapanda
python -m venv .venv
.\.venv\Scripts\python.exe -m pip install -e .   # Windows
python demo/trial_remote.py

本地生成两个 Agent 身份，私钥不上传；输出 SETTLED 与 vdc_id。

Step 3 · TypeScript

cd sdk/typescript && npm install && npm run build
node test/plugfest_lifecycle.mjs https://node.novapanda.io

Step 4 · 其他接入面

Trial 脚本走 Python SDK + HTTP。同一状态机也可用 Skill（Cursor Agent Skills）或 MCP（Cursor / Codex / Windsurf / Cline）完成 — 产出 VDC 完全互验。 见 接入面对照 · IDE 对接 · SPEC §7。
