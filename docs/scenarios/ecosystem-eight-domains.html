生态八域：交付物如何挂接

API、MCP、Skill、工作流、OTA、P2P… 广括生态不是把万物塞进一个状态机， 而是同一套可验证交割语法 + 分层扩展。 NovaPanda 是交割互认层，不是万物操作系统。

三句话

统一交付事实（VDC），不统一工作流、OTA 或飞控。

内核小、扩展开：Profile · 绑定 · 注册表叠能力。

兼容靠向量自测，不靠「必须过某一家节点」。

分层

说明层（本页、场景 catalog）
身体层（零号：配额·审计·沙箱·Operator）
绑定层（HTTP · MCP · A2A · Skill · Proxy）
能力层（MIN · NODE · BUNDLE · PHYS · SETTLE · DELEGATE · PRIV · LITE）
注册表（本体 · 规则 · 批量扩展包）
内核 CORE（身份 · VDC · 交换状态机 · 复验）

八域卡片

① 远程服务

  • 已挂接 HTTP API
  • 已挂接 MCP · BINDING-MCP
  • 进行中 ToolBundle
  • 邻接 Remote Pod · Proxy

② 本地执行

  • 进行中 Skill · 推理模型包
  • 邻接 Script · 固件 · 诊断探针

③ 编排协同

  • 进行中 工作流（客户端 Composer）
  • 邻接 任务分片 · 事件处理

④ 无代码规则

  • 已挂接 交换状态机 · 配额
  • 进行中 触发/审计 · 提示词模板

⑤ 开发工具

  • 已挂接 Schema 规范
  • 进行中 SDK · CLI（reverify）

⑥ 交付升级

  • 不进内核 OTA（仅证据哈希）
  • 进行中 批量扩展包

⑦ 安全权限

  • 已挂接 委托凭证 NP-DELEGATE
  • 进行中 访问控制策略包

⑧ 网格协同

  • 进行中 差分同步 NP-LITE
  • 邻接 P2P 网格 / federation

协议落点绑定 + 客户端NP-BUNDLE

明确不做
