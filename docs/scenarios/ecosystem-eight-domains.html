Ecosystem · 生态角色与边界

谁在里面扮演什么角色

NovaPanda 生态中的 4 大核心角色与 4 大基础支撑——统一交付事实，不统一操作系统。

核心角色

1

交换主体 · Agent

持有 Ed25519 身份，propose / deliver / 双签的真正主体。

2

运营实例 · Node

编排交换、配额与 API；可替代，不是协议本身。

3

可信存证 · Registry

规则、本体与扩展包的公开登记，支撑互认向量。

4

承兑商 · Settler

可选清算轨（mock / x402 / 法币伙伴）——下游翻译层。

基础支撑

5

协议规范 · SPEC

状态机与语义的唯一真相源，CC BY 4.0 公共物。

6

发现协议 · Manifest

/.well-known/novapanda.json——陌生 Agent 瞬间识别能力。

7

凭证格式 · VDC

可验证交割凭证：双签、canonical、可离线复验。

8

验证工具 · Verifier

本地 / 任意实现均可 reverify——兼容靠向量，不靠域名。
