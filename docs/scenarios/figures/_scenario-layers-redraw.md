## 这页要讲什么（读这段再画）

场景页想换一个切面，把**三件事**说清楚：

1. **单笔交换** —— 一次 propose → 交付 → **一张 VDC**  
   （抽取、车充、机器人任务……形态不同，**语法同一**）

2. **叠加 / 嵌套交换** —— 一笔业务多腿、**多张 VDC**  
   （尽调包、到场巡检闭环；`depends_on` / Bundle，**不是一张超级票**）

3. **钱怎么进来**（含法币 / 加密币 / mock） —— 清算是**可选下游**：  
   可无真钱（只证交付）、可接 mock、x402、稳定币、法币伙伴；  
   **货币参与 ≠ 协议本体**，**VDC 仍是第一公民**

**一句话（图的灵魂）**：  
场景页讲的是 **「交割怎么叠、钱在哪一层」**，不是再讲一遍「有哪些智能体」。

愿景页已有「智能万物交换」主体图——**本图不要重复画主体圆环 / Agent·IoT 图鉴**。

## 三栏画什么

### A · 单笔交换

一条短流程即可：

```text
propose → 交付 → 双签 → 一张 VDC（可离线复验）
```

下面用 **3～5 个小形态标签**说明「形态不同、语法同一」（点到为止，勿展开成图鉴）：

- 抽取 · 任务 · Token  
- 车充 · 机器人任务 · …（任选）

角标：`语法同一`

### B · 叠加 / 嵌套交换

必须画出「多张小 VDC」而不是一张大票。两条对照即可：

| 现在可排练 | 灯塔 |
|------------|------|
| 尽调包：VDC₁ 抽取 → VDC₂ 任务 → VDC₃ 摘要 | 到场巡检：车到场 → 无人机 → 机器人 →（可选）充电 |
| `depends_on` / Bundle | 每步一张 VDC |

中间大字：`叠加 = 多张 VDC`

### C · 钱怎么进来

画成 **VDC 在上、清算在下** 的剖面（钱不能画成中心）：

```text
┌─────────────────────────┐
│ 交割层（第一公民）        │
│ VDC · 双签 · 离线复验     │
└───────────┬─────────────┘
            │ 可选接入
┌───────────▼─────────────┐
│ 清算层（下游可插拔）      │
│ 无真钱（只证交付）        │
│ mock · x402 · 稳定币     │
│ 法币伙伴                 │
└─────────────────────────┘
```

角标必须出现：

- `货币参与 ≠ 协议本体`
- `Trial = mock · 无真钱`
