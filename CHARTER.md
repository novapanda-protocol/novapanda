# NovaPanda Charter · 范围宪章

> **状态**：现行 · 2026-07-08  
> **角色**：标准体系五件套之首（CHARTER · SPEC · PROFILES · CONFORMANCE · GOVERNANCE+VERSIONING+SECURITY）。  
> **规范性关键词**：RFC 2119 用于配套 SPEC；本文界定范围与成功判据，本身以陈述为准。

---

## 1. Mission

为**智能体、设备与模型**提供一套与平台无关的**可验证交割语法**：  
在无预建关系下完成跨主体交付互认——第一公民是 **VDC（可验证交割凭证）**，而非账户余额或代币。

## 2. In Scope

- 身份（本地持钥、可验签）与规范化密码学约束  
- 交换状态机与终态退款语义（编排层）  
- VDC 的签发、双签、独立复验  
- 验收接口（给定 deliverable + rule → 确定性可复算结果）  
- 结算**适配语义**（可插拔；协议不持牌、不发币）  
- 发现（Manifest）、接入翻译（HTTP / MCP / A2A / Skill / SDK…）  
- 信誉与可选见证的数据形状  
- **组合交割 Profile**（多笔 VDC + correlation；不替代原子收据）  
- 一致性：可测 MUST、黄金向量、Profile 宣告  
- **生态扩展登记**：远程服务 / 本地执行 / 编排 / 规则 / 工具 / 升级 / 安全 / 网格等交付物类型，经 Profile · BINDING · 注册表挂接（见内部 [`生态交付物八域全景与标准支撑.md`](internal/design/生态交付物八域全景与标准支撑.md)）  

## 3. Out of Scope

- 发行或依赖协议代币；对协议层按笔抽成  
- 银行、支付、清算牌照义务本身  
- 自动驾驶控制栈、飞控、功能安全认证、道路/空域行政许可  
- 强制单一节点、单一运营商或「官方域名即真源」  
- 用运营账户（邮箱/手机）替代 Agent 签名完成交割鉴权  
- 将工作流引擎塞入核心状态机以「支持一切业务」  
- 以设备运行态（空闲/执行/暂停/故障等）**替代或并行冒充**交换状态机  
- 以任一运营节点（含零号）扫描作为实现协议之强制准入条件  

## 4. Design Principles

1. **端到端**：真理由当事方签名与可复验性保证，不信任节点。  
2. **最小内核**：CORE 稳定；能力经 Profile / 注册表 / 旁路增强。  
3. **统一交割语义，行业仅扩展**：IoT / 机器人 / 车 / 无人机 / 软件 Agent 共用同一身份·状态机·VDC 语法；差异进 **本体类型与 Profile**，不进并行状态机。飞控/功能安全仍 Out of Scope。  
4. **算力与联结梯度（演进）**：允许 full / edge / lite 实现档共享语义；弱网可 store-and-forward；极低功耗档经未来轻量 Profile/绑定约束载荷——**不**降低复验数学前提，**不**要求私钥上送网关。  
5. **多实现**：规范与实现分离；兼容靠向量，不靠单一代码库。  
6. **加性演进**：未知字段可忽略；破坏复验语义走大版本 + 迁移。密码学双栈（若引入）须协商宣告并走版本纪律，禁止无向量的「算法开关」。  
7. **能力协商**：实现通过 Manifest 宣告 Profiles，而非隐式假设。  
8. **可退出**：数据可导出；换节点不丢已签 VDC。  
9. **开放无协议层绑定**：宽松许可（规范 CC BY · 代码 Apache-2.0）；自测与实现 **零协议层商业门槛**；正式认证标可选且不作入网闸。运营方身体服务另论，不阻自建。  
10. **辖区中立与合规切分**：协议不绑定单一国家运营真源；隐私最小化（见隐私 Profile 意向）；KYC/AML/个保法等辖区义务由运营方与结算伙伴承担，协议不冒充实合包票。  
11. **传输与载体无关**：HTTP / 窄带 / 卫星 / 工业总线等仅经 **BINDING**；上层状态机与 VDC 语义不变。  
12. **时间与防重放**：交换相关消息应可携带 **UTC** 时间依据；幂等键与时间窗须容忍跨国高延迟；时钟源（NTP/PTP/RTC）为部署选择，不强制单一授时。  
13. **规范语文**：规范性字段名、错误码标识以 **ASCII** 为真源；载荷与日志编码以 **UTF-8** 为准；自然语言描述可 i18n 扩展，不得充当解析真源。  
14. **扩展纪律**：厂商自定义字段 MUST 使用前缀或登记扩展点，MUST NOT 破坏 CORE 复验与加性演进；禁止不兼容私有分叉冒充同名协议。

规范**不**将下列纳入 CORE：无线电频谱合法发射义务、空域/道路通行控制律、功能安全 Fail-Safe 运动指令、多智能体路径冲突仲裁引擎、OTA 宿主协议、各国入网认证发证本身——上述可作为外部规则/证书之 **引用或证据哈希** 出现在条款与 Profile。

## 5. Success Criterion（Litmus）

> 一个从未见过本运营方、使用异语言 / 异框架 / 异结算轨的陌生 Agent，  
> 能否**不依赖我方专有组件、不经许可**，仅凭公开规范完成交割，  
> 并被**任意第三方独立复验**？  

能 ⇒ NovaPanda-compatible（至少满足最小 Profile **NP-MIN**）。不能 ⇒ 不是同一协议。

## 7. Value Circulation（叙事边界）

协议锚定**交付事实（VDC）**；经济权柄经**可插拔结算旁路**（及可选 Claim 转让 Profile）多跳流转。  
**不**将 VDC 本身定义为可转账代币。详见 scenarios 与内部 BA 叙述；规范性转让属未来 Profile。
