# 自营节点：主体、银行与合规（通用说明）

> **非法律意见**。具体以律师、支付伙伴、云厂商当地规则为准。

---

## 一、按阶段：要不要企业 / 银行？

| 阶段 | 做什么 | 企业主体 | 银行账户 | 支付牌照 |
|------|--------|:--------:|:--------:|:--------:|
| **1 mock 节点** | 全球免费/试用交换，无真钱 | **不强制** | **不需要** | **不需要** |
| **2 x402 sandbox** | 接伙伴测试网关，sandbox 币 | **强烈建议** | 视伙伴要求；sandbox 有时个人可开 | **不需要你持牌** |
| **2→3 生产 x402/AP2** | 真钱 capture/void | **需要** | **需要** 收清算服务费/对账 | **伙伴持牌，你是技术方** |
| **3 法币 rail** | 人民币/美元企业结算 | **需要** | **需要** | **持牌伙伴持牌** |

**结论：**

- **先 mock 上云**：可以用 **个人** 试跑技术验证，但 **对外「全球公共服务」** 建议尽快用 **公司或个体户** 签 ToS、开发票、接支付伙伴。  
- **接 x402 sandbox**：多数伙伴 **企业 KYC** 更顺；国内 **公司可以** 申请国际云服务与国际支付 sandbox（视伙伴是否支持中国主体）。  
- **银行账号**：mock **不需要**；要开始 **收服务费、对公结算、伙伴分润** 时需要 **对公账户**。

## 二、国内公司可以吗？

**可以**，且对长期运营通常 **更合适**：

| 事项 | 国内公司 |
|------|----------|
| 在 **AWS / 阿里云国际 / GCP** 部署节点 | ✅ 常见；用公司实名注册云账号、外币/人民币付款 |
| 对外提供 **全球 HTTPS API** | ✅ 服务器在 **境外区域** 时，一般 **不涉及国内 ICP 备案**（网站主要面向海外） |
| 服务器在 **中国大陆** 且对公网提供 Web | ⚠️ 通常需要 **ICP 备案** |
| 签 **用户协议 / 隐私政策** | ✅ 公司名义更可信 |
| 申请 **商标** | ✅ 公司或个人/个体户均可；公司便于 CLA、融资 |
| 接 **x402 / Stripe 类** 国际 sandbox | ⚠️ **视伙伴政策**；部分仅支持特定国家主体，需逐家确认 |
| **自己申请支付牌照** | ❌ 不建议；做法币时用 **持牌机构 API** |

**个体户 vs 有限公司：**

- **个体户**：启动快，商标/域名/小规模运营够用；部分国际支付伙伴可能不接受。  
- **有限公司**：接伙伴、融资、雇人、对公合同更顺；维护成本略高。

《行动计划》里的 **轻量公司** 与「先 mock、后 x402」路径一致：**mock 可先上线验证，接 sandbox 前完成主体** 较稳妥。

## 三、自营节点在法律上是什么？

建议对外表述（ToS 中）：

1. 你们提供 **交换编排、VDC 签发、信誉账本（节点侧）** 的技术服务。  
2. **不是银行、不是支付机构、不托管用户资金**（mock 无资金；x402 资金在网关/链上或持牌方）。  
3. **开源代码** 任何人可自建；**你运营的节点** 是其中一个托管实例。  
4. **信誉数据** 默认 **不随代码开源**（策略见 GOVERNANCE）；用户数据按隐私政策处理。

## 四、接 x402 sandbox 前准备（商务/合规）

- [ ] 企业营业执照（国内公司或境外子公司）  
- [ ] 对公邮箱、官网域名（新品牌定稿后）  
- [ ] 用户服务协议 + 隐私政策（中/英）  
- [ ] 伙伴要求的 **KYC 材料**（业务描述、网站、法人身份证等）  
- [ ] **API Key 保管**：仅环境变量 / 密钥管理，不进 git  
- [ ] sandbox 与 production **环境隔离**（不同 URL、不同 DB、不同 Key）

## 五、mock 全球公共服务仍建议准备

即使 **不要银行账号**，也建议有：

| 项 | 原因 |
|----|------|
| 用户协议 + 免责声明 | 限制责任、说明非金融托管 |
| 隐私政策 | 存 agent_id、交换 metadata、日志 |
| 滥用防护 | 限流、Admin token、sweep 鉴权 |
| 监控与备份 | SLA 与数据安全 |
| 支持邮箱 | `support@` / `security@` |

## 六、常见问题

**Q：个人能先上线 mock 吗？**  
A：技术上可以；对外商用建议个体户或公司，避免 ToS 与伙伴 KYC 卡住。

**Q：必须有境外公司吗？**  
A：不是必须。服务器在境外 + 国内公司运营很常见；若某支付伙伴仅接受 US/EU 主体，再考虑 **境外子公司** 或换伙伴。

**Q：mock 会踩支付监管吗？**  
A：mock 不转移真实价值，风险低；但页面勿宣传「已接入某某银行/已持牌」，避免误导。

**Q：什么时候开始要银行账号？**  
A：向用户 **收清算服务费 / SaaS 费**，或伙伴 **对公结算** 时需要对公账户。
