## 一、缘起与北极星

### 1.1 问题从哪来

未来 AI 若实现自进化，大量**智能体、智能终端、智能设备**将自主发起请求、调用能力、占用稀缺资源。它们之间是否需要一种**交换价值**的介质——类似今天的钱，但可能是另一种形式？

设想方向：**做交换的通道或基础设施**，而不是再造一种全球货币。

### 1.2 为什么可能需要「另一种钱」

| 场景 | 没有价值媒介时的问题 |
|------|----------------------|
| 调用外部 API / 算力 / 数据 | 无法自动议价、限流、防滥用 |
| 跨主体协作（A 规划，B 执行，C 验证） | 无法按贡献分账、无法激励诚实行为 |
| 开放网络上的陌生智能体 | 免费搭车、垃圾请求、DDoS 式调用 |
| 长期协作与信誉 | 只有日志，没有可转让的「信用资产」 |

**结论**：不一定是取代人民币，更像是**机器经济里的结算与准入层**——可编程、可计量、可审计的价值交换形式。

### 1.3 北极星（一句话）

> **智能世界不是先统一货币，而是先统一「交割与互认」；所有智能体与设备在交换稀缺能力时，都经过同一套可验证语言。**

「都经过它」≠ 经过某家公司，而是经过**同一种交换语法**——像数据经过 TCP/IP，而非经过某家 ISP。

你要成为的是智能时代的 **「交割层 IP」**，不是 **「央行 + 支付宝」**。

### 1.4 与钱包路线的分水岭

| | 钱包延伸（AP2/x402 等） | 你要的交割层 |
|--|------------------------|--------------|
| 一等公民 | 钱、授权、余额 | **可验证的交付证明** |
| 成功标志 | 清算额、支付量 | **陌生对端互认一张收据** |
| 组织叙事 | 支付、牌照、金融科技 | 标准、互操作、计量信任 |

## 二、价值分层：从物理到清算

### 2.1 四层价值形态（早期推演）

1. **资源券（最现实）** — 算力秒、token 额度、存储 GB·天；封闭生态内已能用。  
2. **能力凭证（半开放）** — 「完成合规数据清洗并签名」才可兑换某服务。  
3. **信誉质押（防作恶）** — 调用前锁定，失败罚没；类似押金。  
4. **通用交换单位（最难）** — 跨厂商、跨法域、可兑法币；接近真钱 + 牌照 + 反洗钱。

实务上多**混用**：日常资源券，跨域大厂积分，极少数链上或法币通道。

### 2.2 三层抽象（物理 → 交换 → 清算）

| 层 | 是什么 | 例子 |
|----|--------|------|
| **物理层** | 真实稀缺物 | 千瓦时、一次推理、机械动作 |
| **计量层** | 对物理层的测量 | 电表读数、API 日志、结果哈希 |
| **交换层** | 双方认可的**交割证明** | 带双签的二进制凭证 + 规则 ID |
| **清算层**（在上） | 记账与兑现单位 | 钱、token、算力券、合同账 |

**token / 钱 / 算力券**是清算层的事；用来汇总、抵扣、兑现交割证明，不是设备握手时必先谈清的。

### 2.3 「钱」在智能体世界的四种层次（务实判断）

| 维度 | 结论 |
|------|------|
| 封闭生态（自家 Agent 调自家 API） | **已可行** — API Key + 账单 + 配额 |
| 企业 B2B | **可行** — 合同 + 发票 + 托管账户 |
| 开放互联网任意 Agent 互付 | **部分可行** — 监管、欺诈、交付证明是瓶颈 |
| 完全去中心「机器货币」 | **长期探索** — 不适合第一个产品 |

**做交换通道比做全球机器法币更可行**；先做结算与准入，比先造币更落地。

### 2.4 与自进化 AI 的关系

- 进化需要外部资源（数据、算力、人类反馈），必须有预算与约束。  
- 无预算的自主体在开放环境会变成滥用者，而非协作者。  
- 交换介质是**进化速度的调节阀**：贵的能力少被调用，低质量服务被淘汰。

**方向一致**：不是炫技币，而是资源与责任的记账系统。

## 三、可验证交割凭证（VDC）与 GDU

### 3.1 定义

**可验证交割凭证（Verifiable Delivery Claim, VDC）**  
全球标准的结构化记录（常序列化为 CBOR/Protobuf 等二进制），含义固定：

| 字段逻辑 | 含义 |
|----------|------|
| **主体** | 谁交付、谁接收（密码学身份，全球唯一） |
| **资源类型** | 全球统一类型码 |
| **数量与质量** | 多少、精度、SLA、结果摘要/哈希 |
| **规则集** | 规则 ID（非具体价钱） |
| **证据链** | 计量来源、时间窗、可选第三方见证 |
| **证据等级** | 自报 / 双签 / 硬件计量 / 第三方见证 |
| **签名** | 交付方签 + 接收方确认签 |

**完成**的标志是 **VDC 成立**，不是扣款成功。

### 3.2 通用交割单位（GDU）

> 在规则 R 下，主体 A 向主体 B 完成了类型为 T、数量为 Q 的一次可验证交割。

像货运提单或仓单：可转让、可抵押、可批量清算，根子锚在**真实交付**，不是凭空 mint 的积分。

### 3.3 资源类型示例（资源本体 v0）

```
energy.electric.dc
compute.inference.tokens
data.sensor.stream.1h
actuation.robot.grasp
service.task
```

新类型可扩展，须有**注册与版本**，避免各说各话。

### 3.4 规则注册表示例

| 规则 ID | 含义 |
|---------|------|
| `R-grid-2030` | 1 kWh 结算参考 |
| `R-inference-open` | 1M output tokens 参考 |
| `R-barter-strict` | 仅允许同类 1:1 |
| `R-clean` | 外立面清洗验收规则 |

跨类等价（电换推理）写在**规则注册表与市场**，不写在凭证里；交换层不冒充定价上帝。

### 3.5 交换状态机（万物同一仪式）

1. **相遇** — 发现对方能做什么  
2. **缔约** — 在规则 R 下谈妥交什么、多少  
3. **承诺** — 锁定可执行的责任边界（托管/信用额度）  
4. **交付** — 物理或数字世界实际发生转移  
5. **确认** — 双签生成 VDC  
6. **流转** — 可选：转让、抵押、批量清算  
7. **争议** — 申诉、仲裁、罚没  

### 3.6 充电场景：车 ↔ 充电机器人（握手要点）

物理上是电；协议上交换**报文 + 签名**：

1. 能力声明（身份、预算上限、接受的规则集）  
2. 报价（规则 ID、换电比例、最大电流）  
3. 锁定承诺（托管额度 / 信用额度指纹）  
4. 物理充电 + 同步计量  
5. 交割证明（起止读数、ΔkWh、时间戳、Bot 签）  
6. 确认 + 释放（Car 签或联合签）  
7. 可选：上传证明至清算网换法币/token  

逻辑记录字段：`who` / `what` / `how_much` / `under_rule` / `evidence` / `signatures`

**为何需要锁定**：否则充完拒签、虚报电量；完整交换 = **能力锁定 + 物理交割 + 联合签名释放**。

### 3.7 与今日技术距离

| | 已有 | 缺口 |
|--|------|------|
| 车-桩 | ISO 15118、Plug & Charge、后台扣款 | 车-机器人点对点、统一交割凭证、多资产自由折算 |
| 软件 API | x402 按次付 | 以「交付了什么」为一等公民的全球类型码 |

### 3.8 全球普适：能统一什么、不能统一什么

| 可以全球普适 | 很难全球单一化 |
|--------------|----------------|
| 身份表明方式 | 谁发行「真钱」 |
| 交换的二进制结构 | 各国支付牌照、税务 |
| 「交付了什么」的描述 | 所有资源一个标价数字 |
| 计量与签名验证 | 无中心争议终审 |
| 凭证折成当地结算 | 一家公司积分当全球通货 |

全球普适的应是：**统一的交割描述语言 + 统一的交换状态机**，而不是统一的「价值数字」。

## 四、交换通道与全球四层架构

### 4.1 交换通道五要素（MVP 思路）

- **身份** — 可验证 ID（密钥、证书、DID）  
- **目录** — 能做什么、单价、SLA、失败怎么退  
- **计量** — 不可抵赖用量单  
- **结算** — 同步/异步；预付费、微支付、托管释放  
- **信任** — 信誉 + 质押 + 第三方仲裁  

本质：**发现 + 合约化报价 + 计量 + 清算 + 争议**，不是某一种币的名字。

### 4.2 全球四层架构

```
L1 身份与发现（全球）     → 设备/Agent 身份、能力目录
L2 交换协议（全球核心）   → VDC 格式、握手状态机
L3 清算联邦（多机构）     → 折法币/各生态积分
L4 监管与物理（各国/各域）→ 法律、电力市场、真实计量硬件
```

- L1–L2：像 DNS + TLS + HTTP，开放 RFC，多实现。  
- L3：无数清算方，认同一 VDC 输入。  
- L4：永远本地化；协议不能代替法律。

### 4.3 三根全球公用基础设施

1. **全球资源本体（Resource Ontology）**  
2. **全球规则注册表（Rule Registry）** — 子规则须引用父规则 ID  
3. **全球信任根（Trust Anchors）** — PKI/DID、见证节点、联邦声誉日志  

### 4.4 大模型在协议中的位置

大模型 = **带算力与策略的服务端点**，与充电桩无本质区别：提供类型 T 的服务并签发 VDC。

| 交换 | VDC 写法 |
|------|----------|
| 桩 ↔ 车 | `energy.electric` + 电表证据 |
| 模型 A ↔ 模型 B | `compute.inference` 或 `service.task` + 结果哈希 |
| 摄像头 ↔ 分析模型 | `data.stream` + 采样证明 |
| 机器人 ↔ 调度 Agent | `actuation.*` + 完成确认 |
| 任意 ↔ 人类公司 | VDC → 清算枢纽 → 法币/合同账 |

### 4.5 开放互联网式五条设计原则

1. 开放标准（IETF/W3C/ISO 类流程 + plugfest）  
2. 多清算、单协议  
3. 端到端签名（中间只转发，可比 TLS）  
4. 可携带凭证（像支票，非只能在发行商店用）  
5. 无单点 mint（全球层只发行**标准**，不发行通用币）  

### 4.6 概念阶段全球路线（非排期）

1. 定义 VDC 1.0（字段、签名、错误码、幂等、重放防护）  
2. 定义交换状态机  
3. 发布资源本体 v0（能源、算力、推理、数据、动作五类）  
4. 规则注册表开源  
5. 互操作测试网（陌生对端能否成交一张 VDC）  
6. 清算联邦（多家认同一 VDC 入账）  

### 4.7 极窄 MVP 故事（技术验证用）

> Agent A 向 Agent B 购买一次结构化抽取：报价 → 托管 0.01 单位 → B 返回带签名结果 → 自动释放或超时退款。

须想清楚五件事：**单位、发行、交付证明、失败策略、审计**。

## 五、从人类世界到智能世界：转移蓝图

### 5.1 两个世界

**人类世界（正在消退的默认）**

- 人是责任与授权的终点  
- 价值以货币与法律账目为中心：先有钱，再做事  
- 协作靠组织边界  
- 机器是工具，无交换主体性  
- 信任靠品牌、司法、中介  

**智能世界（正在形成的默认）**

- 智能体与设备是协作节点  
- 价值以可验证交付为中心  
- 协作靠共同交换语法  
- 信任靠密码学身份、计量、质押、凭证历史  
- 人类保留终审与例外  

转移是**长时间双层并存**：人类管「谁最终负责」，智能世界管「机器如何交换」。

### 5.2 三次位移

| 位移 | 从 | 到 |
|------|----|----|
| 责任 | 人直接操作一切 | 人定边界，机链式执行，人保留仲裁 |
| 价值锚点 | 余额变动 | 交付记录（VDC） |
| 互联方式 | 平台账号与支付 | 陌生对端也可交换 |

### 5.3 天空层 + 地球层

**天空层（智能世界）** — 命名资源、记录交付、约束交换、翻译接口；**不**铸币、立法、取代法院。

**地球层（人类世界）** — 生命权、财产权、货币财政、反垄断、数据主权、**最终责任追究**。

原则：**天空层越繁荣，地球层越不能缺位。**

### 5.4 转移期双轨

| 轨道 | 作用 |
|------|------|
| **凭证轨** | 车→桩→机器人→推理… 链式交换 |
| **货币轨** | 工资、税、跨国贸易、宏观稳定 |

两轨经**清算接口**连接；无人被强迫每一步都兑现法币。

### 5.5 四个时代（叙事）

| 阶段 | 名称 | 特征 |
|------|------|------|
| 0 | 围墙花园 | API Key、各云额度；交割语义分裂 |
| 1 | 影子层 | 机器记 VDC，人逐步批准 |
| 2 | 双轨层 | 可计量场景凭证轨与货币轨并行 |
| 3 | 默认层 | 新设备默认交换状态机 |
| 4 | 宪制层 | 宪法、规则注册表、见证市场成熟 |

### 5.6 人类角色：操作者 → 宪制者

| 曾经 | 转移中 | 稳定后 |
|------|--------|--------|
| 人刷卡充电 | 人设上限，车桩交换 | 人处理异常 |
| 人雇保洁 | 人定标准，桩与机器人交换 | 人管责任链 |
| 人点 API | 人授权 Agent | 人审规则与高风险域 |

人的尊严：**不可被协议废除的最终否决权**（生命、自由、重大财产）。

### 5.7 必须从人类世界带过去的行李

1. 责任可追溯（凭证可转让，责任链不消失）  
2. 权利与同意（可读、可撤回）  
3. 公平与救济  
4. 货币与财政仍属人类制度  
5. 开放与反垄断（交换语法属共域）  

### 5.8 必须降级或告别的习惯

- 每件事先扫码付钱 → 降为可选兑现  
- 平台内余额不可携 → 凭证与声誉可迁移  
- 服务完成无标准收据 → VDC 为默认输出  
- 机器只是奴隶 → 机器为交换主体（在授权壳内）  
- 黑箱自动化 → 证据等级强制标注  

### 5.9 蓝图原则（自我约束）

1. 交换事实不可篡改  
2. 交换语法不可垄断  
3. 价值译法可以竞争  
4. 永远不做唯一铸币方  
5. 永远先证交付，再谈价格  

### 5.10 风险与暗流

| 风险 | 说明 |
|------|------|
| 凭证封建主义 | 大厂垄断见证与格式 |
| 责任蒸发 | 转手无数次无人负责 |
| 监控国家 | 交换层变全景账本 |
| 双轨撕裂 | 富人凭证闭环，穷人仍靠现金 |
| 伪交换 | 无真实交付的空单 |

抗体：开放语法、证据等级诚实、责任链登记、宪法限权、清算竞争。

### 5.11 愿景三级（考虑用）

- **L1 技术梦**：陌生车↔陌生桩、陌生模型↔陌生模型，无需预建商业关系即可完成锁定→交付→双签。  
- **L2 生态梦**：多清算方认 VDC 入账；AP2/x402 成为下游。  
- **L3 文明叙事**：「智能世界交换」= 先交换交割证明，再交换价值。

## 六、交换链、交换网与三种价值流

### 6.1 最小故事单元

```
车 ←[交换]→ 电桩 ←[交换]→ 清洗机器人 ←[交换]→ 物流 / 模型 / 维修 …
```

**下午 — 车与桩**  
37 kWh，VDC-能源；不必当场人民币。

**傍晚 — 桩与机器人**  
按 R-clean 清洗，证据：影像+水压+时长 → VDC-动作。  
报酬：清算额度、托管释放、或 VDC 拆分转让。

**随后 — 机器人**  
可换充电、耗材、诊断、推理，或打包清算换法币。

**夜里 — 桩与边缘模型**  
传感器片段 ↔ 异常分类 → VDC-推理（同一语法，不同类型码）。

整条链：**没有一环必须先有钱**；每一环要**交付可验证**。

### 6.2 付款链 vs 交换链

**付款链**：车 → 付钱运营商 → 付钱保洁 → 买耗材（钱在中转，交付与支付混杂）

**交换链**：车↔桩↔机器人… 每跳留下凭证；钱/积分仅在需要兑现时出现

### 6.3 三种价值流

| 流 | 说明 |
|----|------|
| **凭证换手** | 互持 VDC；园区、联盟内循环 |
| **凭证抵押** | 先服务，后向清算池兑现 |
| **凭证即信誉** | 不立刻兑钱，用历史 VDC 赢得更难委托 |

### 6.4 交换网（Exchange Mesh）

- **节点**：任何自主智能体/设备  
- **边**：一次成功交换 = 双向确认的 VDC  
- **流**：能源、动作、数据、推理…  
- **汇**：清算枢纽、行业池、监管视图 — 把凭证流**译**为人类账本  

规模化后：凭证市场、信誉维度、规则竞争、局部闭环（园区内不兑现现金）。

### 6.5 与钱包世界对照

| 钱包世界 | 交换世界 |
|----------|----------|
| 每跳默认扣款 | 每跳默认登记交付 |
| 余额是核心状态 | 「完成了什么」是核心状态 |
| 跨主体 = 转账 | 跨主体 = 互签收据 |
| 机器人是付款终端 | 机器人是交换当事方 |

### 6.6 交换链必须留的五条「缝」（宪法位）

1. **跨类怎么抵** → 规则注册表 + 市场  
2. **凭证能否伪造** → 证据等级 + 见证 + 质押罚没  
3. **某环要变现** → 随时进清算层，不强迫全网实时用钱  
4. **失败与争议** → 部分释放、拒签、仲裁槽位  
5. **链条责任** → 人类法域与责任链，技术不能假装没有

## 七、愿景：智能世界的交换

### 7.1 愿景成立的前提（要赌的命题）

| 命题 | 若成立 | 若不成立 |
|------|--------|----------|
| 自主设备/代理爆炸增长 | 跨主体协作刚需 ↑ | 永远大厂围墙 API |
| 协作成本在互信与计量 | 交割凭证比付钱更底层 | API Key + 账单就够 |
| 开放互操作有经济收益 | 厂商愿参加 plugfest | 各自闭环更省钱 |
| 物理+数字可结构化描述 | 资源本体可扩展 | 只剩人类合同 |

不确定性在**经济与政治**，不在「二进制能否表示」。

### 7.2 架构一句话

```
智能体/设备 → [ 全球交割层：VDC + 资源类型 + 状态机 ] → 各清算与合规
```

### 7.3 守住的三个原则

1. 永远不做唯一铸币方  
2. 永远先证「交付」，再谈「价格」  
3. 永远用互操作测试当里程碑  

### 7.4 冷静提醒

「所有智能体、所有设备」是**北极星**；现实路径：

**一个可计量垂直 → 一种收据格式 → 多家清算接口 → 再扩资源类型**

ISO 15118 在车电领域用了十几年；复制到推理、数据、动作，须按**基础设施**时间尺度准备。

## 八、业界现状、洞察与战略选项

### 8.1 直接结论（2025–2026）

| 你的设想 | 现状 |
|----------|------|
| 全球统一交换语言 | 有雏形，分散，无单一赢家 |
| 不绑一家公司 | 名义开放，清算仍靠大厂/支付网 |
| 电/算力/推理/数据/动作同一抽象 | **基本还没有** |
| 比 token 更底层的全球价值单位 | **基本没有**；主流是授权+结算 |

**有人在抢「代理的钱包和收银台」；几乎没人在抢「万物交割语法的 TCP/IP」。**

### 8.2 主要协议与标准

| 名称 | 层次 | 要点 | 与 VDC 关系 |
|------|------|------|-------------|
| **A2A** | 通信 | 发现、对话、委派 | 说话层，不定义交割 |
| **AP2** | 授权/支付 | Mandate 代付；60+ 机构 | 钱是核心 |
| **x402** | 结算 | HTTP 402 按次微支付 | API 计费强 |
| **ACP** | 结账 | OpenAI+Stripe 代理结账 | 电商 |
| **MPP** | 支付会话 | 流式微支付 | 偏支付非全资源 |
| **XAP** | 代理商务 | ExecutionReceipt、VerityReceipt | **概念最接近 VDC** |
| **ERC-8004** | 信任 | 链上身份/声誉/验证 | 非支付 |
| **ISO 15118** | 车电 | Plug & Charge、计量会话 | 能源垂直成熟 |
| **DID/VC** | 身份 | W3C 底座 | 不定义交换 |

**链接备忘**

- AP2：https://github.com/google-agentic-commerce/AP2  
- x402：https://www.x402.org/  
- XAP：https://xap-protocol.org/ | https://github.com/agentralabs/xap-protocol  
- ERC-8004：https://eips.ethereum.org/EIPS/eip-8004  

### 8.3 四条线对比表

| 维度 | ISO 15118 | AP2 | x402 | XAP |
|------|-----------|-----|------|-----|
| 主要交换物 | 充电会话、计量 | Mandate 授权 | 链上支付指令 | 合同+执行收据 |
| 价值一等公民 | 千瓦时 | 法币/授权 | 代币支付 | **交割证明** |
| 典型场景 | 车插电 | 助理代买 | API 按次收费 | 多代理分工 |
| 全球性 | ISO 成熟 | 大厂+支付联盟 | 加密生态 | 早期开源 |
| 人类在环 | 可弱 | **强** | 可弱 | 自主代理 |
| 与「非钱交割」 | **近** | 远 | 远 | **最近** |

### 8.4 六条洞察

**洞察 1**：行业在争「代理商务栈」（对话→授权→付钱→追责），不是「万物价值互联网」。

**洞察 2**：接近你抽象层的是「收据」不是「币」— Mandate、payment proof、ExecutionReceipt、ISO 计量会话；VDC = 后两类泛化到全资源。

**洞察 3**：「全球普适」分裂为两条路 — **A 支付普适**（AP2/x402，有钱有牌照）与 **B 交割普适**（ISO15118/XAP，缺清算联邦）。你的设想是 **B 在上、A 在下**。

**洞察 4**：物理能源走得比纯软件远；MVP 优先**车/桩/机器人**（可计量），纯 API 会先被 x402 占领。

**洞察 5**：信任是隐藏主战场 — 身份、授权、计量、清算、争议；VDC 须声明**计量源等级**。

**洞察 6**：终局是分层粘合 — 通信层卷、授权层卡组织+大厂、结算层碎片、**中间 VDC+资源本体是缺口**。

### 8.5 三条战略选项

| 选项 | 做什么 | 优势 | 风险 |
|------|--------|------|------|
| **① 跟支付栈** | AP2/x402 扩展收据字段 | 流量、联盟现成 | 被当成又一个支付方案 |
| **② 做交割层** | VDC+本体+状态机，对接多清算 | 最贴原始想象 | 冷启动，要等 A 层认 |
| **③ 垂直扎透** | 车电/工业/机器人 | 故事硬、可演示 | 「全球」须慢慢扩类 |

**参考判断**

- 思想验证 / 标准占位 → **② + ③**  
- 商业 2 年内落地 → **③ 为主，① 为接口**  
- 只研究 → 盯 AP2 + XAP + ISO15118 + ERC-8004  

### 8.6 读协议时的三个问题

1. 交换的一等公民是什么？（钱、授权、还是交付证明？）  
2. 陌生对端互认靠什么？（PKI、链、还是平台账号？）  
3. 软件服务怎么证明「已交付」？  

三句都能答清，才与「全球 VDC」同路。

### 8.7 重合与缺口

**已有人做**：可签名授权、按次微支付、协商+收据、跨组织身份、车桩计量会话。

**仍空白**：全球资源本体；交割证明一等公民的开放规范；陌生车↔机器人↔模型 API 同一握手；不依赖单云/单链的清算联邦。

## 九、组织形态：协议、基金会与公司

### 9.1 核心结论

**不必然需要公司以「拥有协议」方式承载**；若认真推进，几乎一定需要某种法人主体（公司、基金会、联合体），**角色不同**。

| 你想拥有的 | 适合载体 |
|------------|----------|
| 开放协议本身 | 基金会 / 标准组织 / 开源社区 |
| 参考实现、测试、认证 | 公司或基金会 |
| 垂直场景落地 | **公司很合适** |
| 清算、发卡、代扣 | 持牌公司（应避开） |

### 9.2 三条路径

**路径 A — 先不注册公司（0→1）**  
GitHub 规范草案 + 参考实现 + plugfest；个人维护即可。  
局限：难签大客户 POC、商标不清、易被疑「收编」。

**路径 B — 基金会/联盟（最贴全球交割层）**  
协议版权/商标放基金会；技术委员会治理；收入来自会员费、认证费，非卖币、非抽每笔交易。

**路径 C — 商业公司**  
定位「主要贡献者 + 工具与集成商」，非协议所有者；赚钱来自参考栈、认证、集成、咨询。  
**不要**每笔 VDC 经公司抽成。

### 9.3 直接问答

| 问题 | 回答 |
|------|------|
| 做交割层必须开公司吗？ | **不必须** |
| 需要某种组织吗？ | **需要**（联合大厂、认证、合同时） |
| 用公司承载全球协议好吗？ | **不好**（中立性） |
| 用公司参与好吗？ | **好**（落地、生计） |

**协议层 → 非营利开放治理；落地与吃饭 → 可以公司；两者分开。**

### 9.4 务实三阶段（组织，非技术排期）

```
阶段 1：无公司也可 → VDC 草案 + 1 个垂直 demo + plugfest
阶段 2：有企业愿试 → 公司签 POC，协议开源
阶段 3：多行业同桌 → 联盟收编规范与认证商标
```

**不要一开始**：注册公司 + 发币 + 做清算。

### 9.5 与钱包路线的组织差异

| | 钱包延伸 | 交割层 |
|--|----------|--------|
| 融资逻辑 | 交易流水 | 生态位、工具、认证、垂直 SaaS |
| 成功标志 | 清算额 | 陌生厂商互认一张收据 |

## 十、宪法序言与根本法提纲

> 宪法管「为什么」和「绝不允许什么」；技术规范管「怎么编码」。

### 10.1 序言（草案）

> 我们承认，智能体与自主设备将构成协作社会的主体网络。  
> 人类不再是每一次交付的操作者，但仍是自由、生命与正义的最终守护者。  
> 因此，我们确立：**交换以可验证的交付为先，以货币兑现为后；交换语法属共域，不属于任一公司；任何权力不得要求隐瞒交付真相，不得以交换之名实施奴役或全景监控。**  
> 智能世界的繁荣，建立在人类世界权利之上，而非替代之。

### 10.2 八条根本法（提纲）

| 条 | 内容 |
|----|------|
| 一 | **交换一等公民** — 交付证明优先于支付记录 |
| 二 | **开放语法** — 核心字段不可专有封锁 |
| 三 | **中立** — 标准组织不清算、不铸币、不与客户争利 |
| 四 | **可验证** — 证据等级不得误导 |
| 五 | **可退出** — 凭证与声誉可迁移 |
| 六 | **人类最终责任** — 生命、重大财产、公共安全保留人工法域 |
| 七 | **不扩张** — 不做思想审查与社会信用唯一来源 |
| 八 | **修正** — 修宪门槛高于改技术规范 |

### 10.3 蓝图级约束（若变成…则失去…）

| 若变成 | 则失去 |
|--------|--------|
| 全球唯一钱包 | 中立性 |
| 全球唯一币 | 政治可能性 |
| 某云专属扩展 | 「都经过它」变成「都经过它家」 |
| 不可审计黑箱 | 设备互认 |
| 每笔必抽成 | 协议被抵制 |

## 十一、附录

### 11.1 术语表

| 术语 | 含义 |
|------|------|
| **VDC** | Verifiable Delivery Claim，可验证交割凭证 |
| **GDU** | Generic Delivery Unit，通用交割单位 |
| **交换层** | 命名、记录、约束、翻译接口；不铸币 |
| **清算层** | 钱、token、算力券；VDC 的译法 |
| **plugfest** | 互操作测试活动 |
| **交割层 IP** | 类比 TCP/IP，非央行+支付宝 |

### 11.2 总纲（可单独引用）

> **从人类世界到智能世界的转移，不是人让位于机器，而是价值从「账户里的数字」迁到「网络上可验证的交付」；人类退到宪法的高度，机器升到交换的密度。智能世界的财富，是累积的、可转让的、可验证的「已经交付」；车、桩、机器人、模型，都是交换网上的节点，而不是收银台前的顾客。**

> **承认交付，先于承认付款；承认交换，联结智能世界。**
