## 二、域名与占坑

| 项 | 状态 | 动作 |
|----|:----:|------|
| 主域名查询（`.org` / `.io` / `.dev` / `.cn` 等） | ☐ | 选定 1 个主域 + 可选 1 个防御域 |
| 主域名注册 | ☐ | 年费占坑，防抢注 |
| GitHub 组织 / 仓库名占坑 | ☐ | 可先 **private**；见 [`BRAND_PREP_LOG.md`](BRAND_PREP_LOG.md) |

| 官方邮箱（`hello@` / `security@`） | ☐ | **有域名后**再设 |

**无域名时的临时方案：** GitHub 仓库 README + `spec/` 作规范入口；避免长期依赖个人 unrelated 域名。

## 四、对外表述红线

**可以说：**

- 开放规范 + 参考实现；VDC 可脱离节点独立复验
- 结算腿可插拔（mock / x402 / AP2 / fiat **HTTP 形状** + fake 测试服务）
- 软件 Agent 模拟舱，排练未来跨主体物理交割
- 与 x402/AP2 **分层**：它们在清算/授权；Troodon 管可验证交割与信誉

**不要说（除非事实已成立）：**

- 「已对接某某持牌清算 / 某某大厂官方 sandbox」
- 「Troodon® 官方认证」——无注册商标与认证流程前
- 「全球标准已发布」——规范 CC BY 可开源，但基金会/认证产品未就绪
- 暗示 x402/AP2/ISO15118 官方背书

## 五、与 x402 / AP2 的分工（对外一页纸）

| 层 | x402 / AP2 等 | Troodon |
|----|---------------|---------|
| 主要问题 | 怎么授权、怎么付钱 | 交付了什么、是否按规则验收、凭证能否复验 |
| 第一公民 | 支付 / 授权意图 | **VDC（可验证交割凭证）** |
| 关系 | 清算/授权 **适配器** | 交换状态机 + 验收 + 信誉；**不铸币、不持牌** |
| 无伙伴时 | 各自生态仍在演进 | 用 **fake gateway** 演示插口，不阻塞协议验证 |

## 七、公开前最低门槛（建议）

**全部满足后再「高调立旗」：**

1. ☐ 文字商标**至少已提交申请**（或律师确认低风险且已占坑）
2. ☐ 主域名已注册
3. ☐ GitHub（或等价）仓库可复现 Quickstart
4. ☐ README footnote 含品牌筹备说明
5. ☐ 本清单第七节表述红线已读

**不必等待：**

- 外部清算 / LLM / 充电桩伙伴
- 图形商标 / 认证徽章
- 公司注册（可与商标并行，非阻塞开发）
