# 治理（Governance）

## 铁律

1. **协议/规范/商标 是公共物，绝不归属任何单一公司。** 公司是"身体"，协议是"灵魂"。
2. 任何人可不经许可地实现规范、自建节点、接入交换——**无许可、可复验、可退出**。
3. 任何治理变更 MUST NOT 破坏既有 VDC 的可复验性与向后兼容的复验路径。

## 分层与归属

| 层 | 内容 | 授权 | 归属 |
|----|------|------|------|
| 规范 spec | 协议文本 + JSON Schema | CC BY 4.0 | 中立组织（过渡期由 steward 代持） |
| 参考实现 code | `novapanda/` | Apache-2.0 | 开源社区 |
| 商标/认证标 | 名称 + "NovaPanda-compatible" 认证标 | 商标法 | 中立组织（过渡期代持） |
| 托管服务 | 运营节点 | 商业 | 公司（"身体"，可被替代） |

## 演进阶段

- **阶段 0 · Stewardship（当下）**：单一 steward（创始人/过渡公司）代持规范与商标，公开决策记录。
  - 代持期承诺：商标仅用于"防止冒名 + 维护认证标公信力"，不得用于排他性收费授权。
- **阶段 1 · 多方维护**：引入外部维护者与 RFC 流程（提案→公示→采纳），变更需多签。
- **阶段 2 · 基金会/中立实体**：将规范著作权与商标转入独立非营利实体，公司退居为众多实现者之一。

## 变更流程（RFC）

1. 以 issue/PR 形式提交提案，标注是否影响**复验语义**（影响者门槛更高）。
2. 公示期 ≥ 14 天。
3. 采纳需维护者达成共识；破坏性变更需提供迁移与兼容复验路径。
